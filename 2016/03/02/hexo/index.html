使用hexo时遇到的小坑

Mar 2, 2016 | hexo | Hits

之前自己搭过一个个人博客，拼凑的前端页面无法直视，再加上后台的编辑器难用的一笔，最后让我没有一点点写博客的动力了…
然后我发现了hexo…简直是不懂前端又想做自己的博客的人的福利啊~
我的动力貌似能重启了~~
markdown也是初学阶段，还花一天时间找了七八个开源的博客系统，没一个满意的。当看到hexo时，突然觉得自己之前真是low的一笔啊,当然经过这个过程也是很锻炼人的……(强行安慰一下自己 :p)

需要准备的:

  • node.js
  • hexo
  • git
  • python(与选择的主题相关，需要注意的是python的版本,不想多说了 - -#)
  • 教程挺多的,安装过程就不说啦。

遇到的坑，帮大家躲一躲：

  • 凡是遇到不是内部命令什么的，不用查了，基本都是因为没有配置环境变量，配置好要关掉窗口再打开才会生效噢…
  • 在配置hexo的_config.yml 时，其实是有两个的，一个是hexo的基础配置，一个是对主题的配置。在配置的时候，属性和值之间必须有个空格的，比如：
    language: zh-CN

  • 当出现error deployer not found:github的时候：(执行hexo d时)
    1).在_config.yml中配置Deployment中，type应该是git而不是github,这是由于版本更新导致的
    2).执行npm install hexo-deployer-git --save然后就好啦~~ 不要问我为什么，因为我也不知道。

  • 下面是安装主题时遇到的问题,那假如你也达到了我这样的审美高度，使用maupassant的话，嘿嘿，下面的坑你需要继续绕（教程）…
    1).需要安装python2.7注意是2.7…
    2).需要安装cnpm。(cnpm是一个完整 npmjs.org 镜像，在天朝安装个应用都要比别人多个步骤~~)
    3).假如你不甚使用了这两个语句:

但没有安装成功的话，首先你需要卸载才可以继续用cnpm继续安装的:
npm uninstall hexo-renderer-sass --save不然还会出错~
如果你忽略这个错误，那么等你部署上去的时候，你会发现你的网站没有样式 - -#

结束:


有时候，我们害怕走错，总是在怀疑，我是不是走歪了，费劲儿了。其实认定了，走就好了，哪怕中途发现错了再去改回来也不迟。至少费劲儿了，也能锻炼出一些肌肉来。
